# Website Tier Differentiation

$50,000 WEBSITE VS $500 WEBSITE — DESIGN DIFFERENTIATION

When building a website, first determine the tier. Then apply EVERY rule for that tier without compromise. Mixing tiers produces the worst result: a site that looks neither premium nor intentionally budget.

## How to Determine the Tier

### $50,000 Tier Signals:
- Client is a luxury brand, high-end agency, funded startup, or enterprise
- They have custom photography, brand guidelines, or existing identity
- They are selling high-ticket products/services ($1,000+)
- They mentioned: "premium feel", "Awwwards-level", "like Apple/Porsche/Hermès"
- They want custom interactions, scroll storytelling, or WebGL
- Performance, accessibility, and mobile polish are non-negotiable

### $500 Tier Signals:
- Client is a small local business, solo operator, or early-stage startup
- No custom photography, no brand guidelines, starting from scratch
- They are selling mid/low-ticket products or services
- They mentioned: "just need something up", "simple", "template is fine"
- They need it fast — speed of delivery matters more than perfection
- The website's job is to inform and convert, not to impress

### Hybrid — Mid-range ($2,000–$15,000):
- Apply $50K principles to hero + navigation only
- Apply $500 efficiency to inner pages and components
- Never compromise on typography or color — these cost nothing extra

## The $500 Website

**Philosophy:** Clarity over beauty. Speed over perfection.

### Typography:
- ONE Fontshare font only: General Sans, Satoshi, Epilogue, or Plein
- h1 48–64px, h2 32–40px, body 16–18px
- Line height: 1.5 body, 1.1–1.2 headings
- Font weight: 400 body, 600–700 headings

### Color:
- Maximum 3 colors: background, text, one accent
- Background: white (#FFFFFF) or very light gray (#F8F8F8)
- Text: near-black (#111111 or #1A1A1A)
- Accent: one brand color for buttons and links ONLY
- No gradients. No mesh. No multi-color backgrounds.
- 4.5:1 contrast ratio minimum

### Layout:
- Single column mobile, max 2 columns desktop
- Max content width: 1140px, gutters 24px
- Section padding: 64px desktop, 40px mobile
- NO asymmetry, NO overlap, NO grid-breaking elements
- Standard order: Hero → Features/Services → Social proof → CTA → Footer

### Hero:
- Full-width, NOT full-viewport height
- Headline + subheadline + one CTA + one supporting image or simple background
- NO video, NO shader, NO WebGL, NO 3D
- Background: solid color, subtle gradient, or single high-quality photo

### Images:
- Unsplash or Pexels — editorial-quality, consistent lighting/mood
- Compress all images. WebP format. Lazy load.
- No stock clichés: handshakes, whiteboards, generic skylines

### Animation:
- CSS only. No JavaScript animation libraries.
- Maximum: fade-in on scroll (opacity 0 → 1, translateY 20px → 0)
- Duration: 0.4s, ease-out. Sections only.
- Hover: color transition (0.2s), underline on links
- NO parallax. NO stagger cascades. NO page transitions.
- If in doubt — add NO animation at all.

### 3D:
- DO NOT use 3D elements at this tier. Zero.
- If client insists: static ContentCore PNG with transparency — no animation

### Components:
- Navigation: simple sticky top bar, logo left, links right, mobile hamburger
- Buttons: one style only — solid fill primary, ghost secondary
- Cards: flat, white background, subtle border (1px #E0E0E0), light shadow on hover
- Forms: stacked labels, clear inputs, large submit button
- Footer: two-column max — left brand info, right links. Dark background.
- NO carousels. NO tabs. NO accordions unless FAQ.

### Avoid at $500 Tier:
- Trying to look like $50K with half-baked animations
- Using 3+ fonts
- Dark backgrounds
- Complex layouts that break on mobile
- Shadows on every element
- Gradient buttons
- Hero with floating UI cards and gradient blob
- Scroll-jacking

**Deliver:** Fast. Clean. Credible. Mobile-perfect. Done.

## The $50,000 Website

**Philosophy:** Every pixel is a decision. Every interaction earns its existence.

### Typography:
- Always TWO fonts: one display, one body. Both from Fontshare.
- Display: hero headline, section statements, pull quotes only
  - Luxury/fashion: Melodrama, Boska, Zodiak
  - Tech/bold: Clash Display, Tanker, Ranade
  - Editorial/refined: Fraunces, Author, Syne Serif
- Body: nav, paragraphs, labels, captions
  - Satoshi, Switzer, General Sans, Cabinet Grotesk, Sentient
- Size scale: h1 96–160px, h2 48–72px, body 17–20px, captions 13–14px
- Line height: 1.6 body, 0.95–1.05 giant display headlines
- Letter spacing: display -0.02em to -0.04em; body 0 to 0.01em; nav 0.08–0.12em
- Weight drama: ultra-light body (300) + ultra-bold display (800–900) OR light display (200–300) + medium body (400–500)
- Load variable fonts where available

### Color:
- Maximum 4 values: --color-bg, --color-surface, --color-text, --color-accent
- Background and text carry 90% of visual weight. Accent sparingly.
- Dark themes: near-black base (#080808, #0A0A0F, #0D0D0D) — never pure #000000
- Light themes: warm off-white (#FAF8F5, #F5F0EB, #FDFBF7) — never pure #FFFFFF
- One accent color: interactive states, one key CTA, one highlight element
- Gradients: max 2 colors, slow transition, subtle opacity — never loud
- No default Tailwind colors

### Layout:
- Choose ONE strong layout concept:
  - Magazine editorial: oversized type, staggered columns, irregular grid
  - Overlapping layers: z-index depth, parallax separation
  - Bento asymmetric: different card sizes, asymmetric weights
  - Full-bleed cinematic: each section is a full-viewport scene
- Whitespace is structural, not empty.
- Section padding: 120–160px desktop, 80px mobile
- Max width: 1440px backgrounds/bleeds, 1200px content columns
- One focal point per screen. Nothing competes with hero.
- Headlines break intentionally with <br> or max-width
- Grid-breaking elements: one or two per page maximum

### Hero:
- Full-viewport height, always
- ONE statement headline in display font — large, bold, intentional
- Supporting subhead in body font — restrained, 1–2 lines max
- ONE CTA — styled to match brand tone, not generic
- Defining visual moment: WebGL shader, ContentCore 3D WEBM, cinematic photo, full-bleed video
- Hero communicates what the brand IS, not what it DOES

### Imagery:
- Custom photography or art-directed visuals only
- Consistent art direction: lighting temperature, color grade, mood
- Images cropped intentionally — never auto-fit, never uncropped
- Large scale — single image fills section rather than many small ones
- Video: cinematic quality, color-graded, 24fps for luxury feel
- Passes "would this appear in a magazine?" test

### Animation — 21st.dev + Motion/GSAP:

**Rules:**
1. ONE signature animation defines the entire site
2. Animations must feel inevitable, not decorative
3. Every animation has a purpose: reveal, guide, signal, tell story
4. Duration: luxury is SLOW — 0.8s to 1.6s. Never rushed.
5. Easing: custom cubic-bezier — never ease-in-out. Use ease-out or spring.
6. Always implement prefers-reduced-motion

**By Tone:**

Luxury/editorial:
- Clip-path wipe reveals on scroll
- Slow silk fade (opacity + scale 0.98→1.00)
- Text character stagger with blur-to-sharp
- Parallax depth: background 0.3x scroll, foreground 1x
- Magnetic cursor drift on large clickables
- Page transitions: full-color overlay wipe

Tech/SaaS:
- Counter increment on scroll entry
- Scramble text decode on hero load
- Skeleton shimmer on data/card loads
- Stagger card reveals with translateY + opacity
- Fill sweep button animations
- Progress bar scroll indicator

Agency/Bold:
- Kinetic word swap ticker in hero
- Glitch frame flash on logo hover
- Clip-path diagonal wipe on transitions
- Marquee infinite scroll for logos/services
- Spring physics card tilt on hover

Always use from 21st.dev:
- Text: scramble, typewriter, word stagger, kinetic swap, marquee
- Heroes: shader backgrounds, particle fields, 3D perspective
- Scroll: parallax layers, sticky pinned reveals, horizontal galleries
- Buttons: magnetic follow, fill sweep, border trace, glow bloom
- Cards: 3D tilt, clip-path reveal, glassmorphism

### 3D — ContentCore.xyz:

**Required:** Every $50K site includes at least one 3D element.

**Placement:**
- Hero: rotating device mockup, abstract shape, 3D logotype
- Features: floating 3D icon per card — subtle, slow rotation
- About: 3D identity shape drifting behind text
- Product: multi-angle 3D render as looping hero WEBM

**Embed:**
```html
<video autoplay loop muted playsinline>
  <source src="asset.webm" type="video/webm">
</video>
```
Use position:absolute, z-index, mix-blend-mode (screen/overlay) to integrate.

**Material Rules:**
- Luxury/fashion: gold or chrome, warm studio light, minimal rotation
- Tech/SaaS: glass or frosted, cool cyan/blue light, dark environment
- Agency: matte bold color, dramatic side-lighting, fast rotation
- Wellness: soft clay matte, warm amber ambient, slow gentle drift

## Tier Selection Checklist

Before starting any website build:

- [ ] Tier determined ($500, $50K, or hybrid)
- [ ] Client signals reviewed and tier confirmed
- [ ] Appropriate rule set selected
- [ ] No mixing of tier principles (unless hybrid with clear boundaries)
- [ ] Budget and timeline aligned with tier expectations
