================================================================================
                      HOW ANIMATIONS WORK
================================================================================

When user scrolls down:

1. Element enters viewport (10% visible)
   ↓
2. Intersection Observer detects it
   ↓
3. Adds "active" class to element
   ↓
4. CSS animation triggers automatically
   ↓
5. Smooth 0.7s fade/slide/scale animation plays
   ↓
6. Element appears elegantly

STAGGERING (Cards cascade in):
  1st card: 0s delay   → animates immediately
  2nd card: 0.1s delay → animates slightly after
  3rd card: 0.2s delay → continues sequence
  Rest:    0.3s delay  → all others together
  
Result: Organized, professional cascade (not chaotic)

================================================================================
                         ANIMATION TYPES
================================================================================

FADE IN
  From: opacity 0, y+30px (below)
  To:   opacity 1, y+0px (normal position)
  Used: Section titles, headings

SLIDE LEFT
  From: opacity 0, x-30px (left side)
  To:   opacity 1, x+0px (normal position)
  Used: Left-aligned cards

SLIDE RIGHT
  From: opacity 0, x+30px (right side)
  To:   opacity 1, x+0px (normal position)
  Used: Right-aligned cards

SCALE IN
  From: opacity 0, scale 0.95 (small)
  To:   opacity 1, scale 1.0 (normal)
  Used: Images, products, team members

EASING
  Timing: 0.7 seconds (professional speed)
  Curve:  cubic-bezier(0.25, 0.46, 0.45, 0.94)
  Feel:   Smooth deceleration, natural motion
  Style:  Like Apple, Google, Stripe
